## 🔴 КРИТИЧЕСКИЕ

### T-004 | Политика видимости событий
**Категории:** 🎯 Архитектура, 👤 Пользовательский опыт, 🔒 Безопасность
**Статус:** Активна
**Почему критично:** Личные события из GEDCOM (рождения, смерти, личная история) сейчас не изолированы от других пользователей. `EventPolicy` резолвит `scope.all` — потенциальная утечка приватных данных.
**Текущее состояние:** Частично. Категоризация (`person/world/country/local`) есть, но Pundit policy не применяет фильтрацию по владельцу.
**Подзадачи:**
- [ ] Добавить `user_id` или `visibility` к событиям типа `person`
- [ ] Исправить `EventPolicy::Scope` — показывать личные события только владельцу
- [ ] Гарантировать, что таймлайны других пользователей не утекают личные факты
- [ ] Написать request-тесты на изоляцию данных

### T-001 | Отслеживание процесса парсинга GED-файла
**Категории:** 👨‍💻 Разработка, 👤 Пользовательский опыт
**Статус:** Активна
**Почему критично:** Пользователь загружает файл и не знает, обрабатывается ли он. Нет никакого feedback'а — ни ошибок, ни прогресса. Это блокирует доверие к продукту.
**Текущее состояние:** Не реализовано. Модель `GedcomFile` не имеет полей статуса.
**Подзадачи:**
- [ ] Миграция: добавить `status` (enum: pending/processing/done/failed), `parsing_started_at`, `parsing_finished_at`, `error_message`
- [ ] Обновлять статус в `Gedcom::UploadWorker` — начало, успех, ошибка
- [ ] Отображать статус в GedcomFiles/Index: иконки, дата последней обработки
- [ ] Зависит от T-002 (оценка FSM) — можно сначала сделать простой enum

### T-013 | Workflow для неизвестных дат
**Статус:** ✅ Выполнено
**Факт:** `FuzzyDate` поддерживает 9+ типов дат: `about, before, after, estimated, calculated, between, from_to, year, month_year`. GEDCOM-модификаторы (`ABT`, `BEF`, `AFT`, `EST`, `CAL`, `BET...AND`) полностью обрабатываются в `DateParser`.
